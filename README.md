# LLM Hidden State Analysis ― 教材と再現コード

AIの「頭の中」（hidden state）を覗くと何が見えるか。**出力の文章を読まずにLocal LLMの一貫性を評価する**という応用までを、入門と応用の二層で扱う教材です。

## 🎯 教材は二層構成です

📘 **[LECTURE_hidden_state.md](LECTURE_hidden_state.md)** ― **本編（入門・全10スライド）**

予備知識ゼロで読める講義資料。「同じ質問なのにAIの答えがゆれるのはなぜ？」という誰もが体験した謎から始め、サイコロの鎖 → 実際に12回聞いてみる → 頭の中を点にして地図で見る（toorPIA）→ 一貫性≠正しさ → AIの得意な土俵マップ → AI合議の夢、までを実物の実験だけで進みます。数式・専門用語は使いません。

📗 **[ADVANCED.md](ADVANCED.md)** ― **応用資料（全19スライド）**

本編の内容を研究レベルまで深掘り：測定の設計（出だし数語の窓・アンカー規格化・d′安全弁）、検証（意味的一貫性との相関）、専門的Q&A（temperature=0でよいのでは？／木展開で直接計算できるのでは？／どの層で測るべきか？）。本編で「自分もやりたい」と思った人の次の一歩。

📙 **[STANCE_CONTINUITY.md](STANCE_CONTINUITY.md)** ― **派生研究：hidden state 軌跡と立場の連続性**

回答の**全トークン**の hidden state を toorPIA で地図化する派生スレッド。中心成果は **「立場（賛成↔反対）の強弱が内部状態に“連続グラデーション”として刻まれている」**（Qwen3-4B・5段ペルソナ／Spearman=0.91）。立場をスイープすると地図が連続的に明滅する**アニメーション**つき。あわせて「分離は内容の意味的距離で決まる」（別トピック→ARI=1.00、同一トピック内→部分的）を横断検証し、**LLM品質評価基準**への展望を述べる。

📕 **[LAYER_MORPH.md](LAYER_MORPH.md)** ― **派生研究：層ごとの内部表現とゆらぎ**

内部表現が **浅い層→深い層** へ進むにつれて立場を **像として結んでいく**様子を、全36層の toorPIA 地図を繋いだ**モーフ・アニメーション**で見せる。さらに、各立場の **代表回答（medoid）** とそのまわりの **“揺らぎ”（束の広がり）** を定量し、**「中立が最も揺れる（確信した立場ほど内部は締まる）」**という、テキストでは見えない**決断性・確信度**の指標を提示する。

## 🔭 研究の射程 ― 自由生成出力の「科学的」評価に向けて

本研究の出発点は、ひとつの課題認識にある。**大規模言語モデル（LLM）の自由生成出力に対する評価は、いまなお“科学的”な評価 ―― すなわち外部判定への依存を最小化し、対象そのものの定量的・再現可能な特性に基づく評価 ―― からは遠い。** 現状の主流は別の LLM や人手の選好に依拠する外部判定であり、判定器への依存性・再現性・循環性の点で、自然科学が前提とする客観性や反証可能性を十分には満たさない。

本研究は、この間隙を **内部状態（hidden state）に根ざした内在的な特性評価** によって埋めることを目指す。toorPIA による内部状態の可視化を通じて見えてきたのは、**出力を粗視化して「軌跡（contour）」として捉えると、その軌跡の“揺らぎ”を、内部表現に基づく定量的特性として評価しうる**という可能性である。当面の中心目標は二つ ―― (1) この内在的特性を **外部判定に依らず再現可能に** 定義・測定すること、(2) その特性が **LLM の性能・振る舞いと相関する** ことを実証的に確かめること ―― にある。

粗視化による軌跡化は、他分野で確立された方法論を導入する道も開く。最も直接的には、**高分子物理学の特性量**（持続長・剛性・慣性半径など、worm-like chain モデルに由来する記述子）を軌跡の集団へ転用できる。さらに将来的なアナロジーとしては、**確率過程論**による定式化や、物理学における**経路積分・最小作用の原理**のような枠組みの援用も視野に入る。これらは現時点では探索的な見通しであるが、**LLM の挙動を、品質判定とは独立な、物理由来の再現可能な言語で記述する**こと ―― その科学的方法論への一歩となることを企図している。

> 本節は確立された結論ではなく、研究の動機と射程を述べたものである。内在的特性の有効性は、正誤・品質との対応づけ（較正）を通じて実証的に検証される必要がある。
