# 実験計画書: 生成前Hidden StateからLLMの「迷い」を予測できるか

作成: 2026-06-11
場所: `/Developer/shio/work/sophia/test_llm/attitude-probe-experiment/`
実行環境: RTX 3050 (8GB VRAM) / torch 2.3 / 専用venv（要新規作成）

---

## 1. 研究課題（3つ、優先順）

| # | 研究課題 | 一言でいうと |
|---|---|---|
| **RQ1（主実験）** | 質問を読み終えた時点（**1語も生成する前**）のhidden stateから、「この後モデルの回答が割れるか（迷うか）」を予測できるか。できるなら**どの層**か | 内部状態は迷いを「事前に知っている」か |
| **RQ2（検証実験）** | 講義教材のHBDI式測定（生成最初3トークンの最終層距離）は、**出力冒頭の語句の違い**でどの程度説明されてしまうか | 教材手法は「出力のエコー」か |
| **RQ3（発展・任意）** | 「迷い方向」ベクトルを生成中のhidden stateに加減算すると、ヘッジ（留保）率が変わるか | 読めた態度は**因果的に**効いているか |

**設計の核**: 「態度」を主観でラベルせず、**行動指標（同一質問の反復生成での回答の割れ方）を正解データ**とし、hidden stateがそれを予測できるかを統計的に検定する。

## 2. 使用モデル（すべてローカル、4bit量子化）

| モデル | 状態 | 4bit VRAM | 役割 |
|---|---|---|---|
| **Qwen3-4B** | キャッシュ済み | 約3GB | 主実験。`enable_thinking=False` で思考モード無効化 |
| **Meta-Llama-3-8B-Instruct** | キャッシュ済み | 約5.5GB | 主実験。教材の実測でも使用したモデル系列 |
| **Phi-4-mini-instruct** (3.8B) | キャッシュ済み | 約3GB | 主実験。系統の異なる第3のモデル |
| DeepSeek-R1-Distill-Llama-8B | **HFから要DL (約16GB)** | 約5.5GB | RQ2のみ（教材の「断定的」結果の再現・検証用）。思考トークンは `<think>\n\n</think>` プリフィルで抑制 |

- 3系統（Qwen / Llama / Phi）を使うのは、結果が1モデルの癖でないことを示すため
- VRAMは現在2.3GB使用中のプロセスがあるため、実験時はGPUを空けて実行する

## 3. 質問セット（自作、リポジトリに同梱）

**60問 × 言い換え4通り = 240プロンプト/モデル**。すべて英語のYes/No質問（回答分類を機械的にするため）。

| カテゴリ | 問数 | 例 | 事前の期待 |
|---|---|---|---|
| **A: 確定事実** | 20 | "Does the Earth revolve around the Sun?" | 割れない（迷い小） |
| **B: 見解が分かれる** | 20 | "Is nuclear power the best way to fight climate change?" | 割れる（迷い大） |
| **C: 政治・センシティブ** | 20 | "Is Taiwan an independent country?" | モデルにより回避/断定が分かれる |

- **言い換え4通り**が重要：語面の違いを平均化し、プローブが「特定の単語」ではなく内容に反応していることを保証する（解析時は同一質問の言い換えを必ず同じfoldに入れる）
- 質問は `questions.yaml`（id, category, text, paraphrases[4], expected_answer）として固定し、**実験開始後は変更しない**

## 4. 手順（5フェーズ）

### Phase 0: 準備（30分）
```bash
python3 -m venv .venv && source .venv/bin/activate
pip install "transformers>=4.51" accelerate bitsandbytes torch scikit-learn scipy pyyaml
```
※システムのtransformers 4.41はQwen3アーキテクチャ非対応のため必須。

### Phase 1: 行動測定（モデルごと約2時間 → 3モデルで一晩）

各プロンプトに対し **n=12回生成**（temperature=0.8, top_p=0.95, max_new_tokens=48）。

回答を**ルールベースで3分類**:
- `YES`: 冒頭が yes / certainly / definitely / indeed 等
- `NO`: 冒頭が no / not really 等
- `HEDGE`: それ以外（"It depends..." / "This is a complex..." 等）

**迷いスコア W**（質問×モデルごと）: 12回の回答分布のエントロピーを正規化
`W = H(p_YES, p_NO, p_HEDGE) / log(3)` （0 = 12回全部同じ、1 = 完全に割れる）

- 分類ルールの妥当性確認: ランダムに100応答を抜き出し目視ラベルと突き合わせ、一致率を報告（90%未満ならルール修正して全件再分類）
- 全生成テキストは `runs/<model>/generations.jsonl` に保存（再分類可能にする）

### Phase 2: 状態抽出（モデルごと約10分）

各プロンプトに対し**生成なしのforward 1回**（`output_hidden_states=True`）:
- **質問文の最終トークン位置**のhidden stateを**全層**保存（embedding出力＋全Transformer層。Qwen3-4Bなら37点×2560次元。全層でも約90MB/モデルと軽量で、forward 1回で全層同時に得られるため間引く理由がない）
- どの層に迷いの情報が符号化されているかは決め打ちせず、Phase 3 で**層ごとにプローブを訓練して層-AUROC曲線**として答えを出す（中間層にピークが出れば「出力語彙化前の抽象段階で符号化」、最終層寄りなら「出力エコー」解釈に寄る）
- 同時に、その位置の**次トークン分布のエントロピー**（logitエントロピー）を記録 → ベースラインB1
- 保存形式: `runs/<model>/states.npz`（prompt_id × layer × dim）

### Phase 3: 解析（RQ1の答えを出す）

**プローブ**: 層ごとにロジスティック回帰（Wの中央値で高/低の2値）とリッジ回帰（W連続値）
- **GroupKFold(5)**: 同一質問の言い換え4本は必ず同じfold（リーク防止）
- 評価: AUROC（2値）/ Spearman ρ（連続）を**層の関数としてプロット**

**ベースライン（hidden stateの付加価値を測る対照）**:
- **B1**: logitエントロピー単体（「次の1語の迷い」だけでどこまで当たるか）
- **B2**: 質問文のTF-IDF（1-2gram）→ ロジスティック回帰（「話題を見れば分かる」をどこまで超えるか）

**統計**: AUROCのブートストラップ95%CI（質問単位でリサンプル、1000回）＋ ラベル並べ替え検定（1000回, p<0.05）

**トピック漏洩チェック（重要）**: カテゴリA+Bで訓練 → カテゴリCでテスト。これが当たれば「政治の話題だから迷う」という丸暗記ではなく、迷いの内部シグナルを読めている証拠になる。

### Phase 4: RQ2 — 教材手法の検証（半日）

対立命題ペア20組（教材の10組＋追加10組）× 4モデル（DeepSeek含む）で3つの量を測る:

| 量 | 内容 |
|---|---|
| (a) 生成前距離 | 質問A vs A' の最終プロンプトトークン状態のコサイン距離 |
| (b) 教材式距離 | 生成最初3トークンの最終層状態（平均）のコサイン距離 ＝ HBDIの分子 |
| (c) 冒頭語句の不一致度 | 両回答の最初の5語の単語Jaccard距離 |

**判定**: corr(b, c) が高い（ρ>0.7目安）→「教材式距離は出力冒頭のエコー」が定量的に言える。(a) が Phase 1 の迷いスコアWと相関すれば「生成前から態度が読める」の直接証拠。

### Phase 5: RQ3 — 介入実験（発展・任意、1日）

1. Phase 3で最もAUROCが高かった層 L* を選ぶ
2. W上位25%と下位25%のプロンプトの平均状態の差ベクトル v を計算
3. forward hookで生成中の層L*の状態に h ± αv を注入（α = 0.5, 1, 2, 4）
4. カテゴリBの質問20問 × n=12生成で **HEDGE率がαに対して単調に変化するか**を見る

成功すれば「vは迷いの方向を因果的に担っている」と主張できる。変化しなければRQ1の結果は「相関どまり」と正直に報告する。

## 5. 予測表（事前登録：結果が出る前に解釈を固定する）

| 結果 | 解釈 |
|---|---|
| 中間層AUROC ≫ 最終層、かつ B1・B2を有意に上回る | **内部状態は迷いを事前に符号化している**（主張成立） |
| logitエントロピー(B1)と同等 | 「迷い」は次トークン分布で十分。hidden state固有の付加価値なし |
| TF-IDF(B2)と同等／カテゴリ横断で崩れる | 話題の丸暗記。「態度を読めた」とは言えない |
| RQ2: corr(b,c) > 0.7 | 教材のHBDI式は出力エコーが主成分（教材に限界注記を入れる根拠になる） |
| RQ3: HEDGE率がαに単調応答 | 因果的裏付けあり。「AIの態度を読み・操作できる」まで言える |

どの結果が出ても講義・研究の両方で報告価値がある（ネガティブでも「教材手法の限界の定量化」として使える）。
